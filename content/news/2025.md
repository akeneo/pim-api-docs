## November 2025

### <span class="label label-news label-added">ADDED</span> Our Postman collection is now based on the OpenAPI specification!

We are excited to announce that our Postman collection is now automatically generated from our `OpenAPI 3.1 specification`. This means you can now test and explore the Akeneo PIM API directly in Postman with up-to-date endpoints, parameters, and examples.

The collection includes all available API endpoints with pre-configured request examples, query parameters, and detailed descriptions to help you get started quickly.

Download the [Postman collection](/files/akeneo-postman-collection.json) and don't forget to also download the [environment template](/files/akeneo-postman-environment.json) to configure your API credentials!

### <span class="label label-news label-added">ADDED</span> Retrieve attributes types on Asset GET

In order to enhance the user experience with our external API, we've added a new property to the [Asset GET routes](/api-reference.html#get_assets): You will now find the `attribute_type` property, which indicates the attribute type for each asset value.  

Our clients always had to make a GET request for the asset family attributes to obtain the attribute type for each asset value. Now, they simply require a single GET request for the asset, reducing the number of API calls needed and increasing productivity.

## October 2025

### <span class="label label-news label-added">ADDED</span> Our new API reference based on the OpenAPI specification is live!

We are excited to announce the release of the new API reference documentation based on our `OpenAPI 3.1 specification`.

This allows developers to be up-to-date in real time on improvements made to the PIM.
This new documentation only applies to the SaaS version of the Akeneo REST API.

The [old documentation](/api-reference-paas-index-70.html) remains available for customers running the PaaS and community versions.
